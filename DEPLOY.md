# 길드 공용 꽃 도감 — 배포 가이드 🌸

길드원끼리 **같은 도감에 함께 입력**하고, **길드 비밀번호**로 외부인 입력을 막는 방법입니다.

작동 방식 한 줄 요약:
- **보기(꽃 보기)** — 누구나 가능 (링크/이미지 공유용)
- **입력(내 꽃 입력)** — 저장하려면 **길드 공통 비밀번호** 필요
- 데이터는 서버의 `data/members.json` 한 곳에 모임 → 모두가 같은 도감을 봄

---

## ⚠️ 가장 중요한 점 두 가지

1. **반드시 비밀번호를 바꾸세요.** 기본값은 `changeme` 입니다. 배포 시 환경변수 `GUILD_PW`로 원하는 비번을 지정하세요.
2. **데이터 보존(영구 디스크)이 필요합니다.** 무료 호스팅은 보통 서버가 잠들거나 재시작할 때 파일이 초기화돼서 **입력한 데이터가 사라집니다.** 그래서 아래 가이드는 *영구 디스크*를 붙이는 단계를 포함합니다. (월 몇 천 원 수준)

## 추천: Render.com 으로 배포 (GitHub 연동, 가장 쉬움)

### 0) 준비물
- GitHub 계정
- 이 프로젝트 폴더(`cozyflowerDB`)를 GitHub 저장소에 올리기 (push)

### 1) Render 가입 & 서비스 생성
1. https://render.com 가입 (GitHub로 로그인하면 편함)
2. 대시보드에서 **New +** → **Web Service**
3. 이 프로젝트 GitHub 저장소 선택 → **Connect**

### 2) 서비스 설정
| 항목 | 값 |
|---|---|
| **Language / Runtime** | Node |
| **Build Command** | (비워두기 — 설치할 의존성 없음) |
| **Start Command** | `npm start` |
| **Instance Type** | 데이터 보존을 위해 유료(예: Starter) 선택 *(무료는 데이터가 사라짐 — 아래 참고)* |

> 포트는 Render가 자동으로 넣어주고(`PORT` 환경변수), 코드가 이미 그 값을 읽습니다.

### 3) 영구 디스크 추가 (데이터 보존) — 꼭!
1. 서비스 설정의 **Disks** → **Add Disk**
2. **Mount Path**: `/var/data`  (크기는 1GB면 충분)

### 4) 환경변수 설정 (Environment)
| Key | Value | 설명 |
|---|---|---|
| `GUILD_PW` | (원하는 비밀번호) | 길드원에게 알려줄 입력용 비번 |
| `DATA_DIR` | `/var/data` | 공용 데이터를 영구 디스크에 저장 |
| `SHARES_DIR` | `/var/data/shares` | 공유 링크 데이터도 영구 디스크에 |

### 5) 배포
- **Create Web Service** 누르면 자동 배포 → `https://<이름>.onrender.com` 주소가 생깁니다.
- 이 주소를 길드원에게 공유하세요.

## 길드원 사용법
1. 사이트 주소 접속
2. **내 꽃 입력** 탭
3. **닉네임** + **길드 비밀번호** 입력 → 가진 꽃 체크 → **저장하기**
4. **꽃 보기** 탭에서 모두의 꽃이 합쳐져 보임

비밀번호는 그 브라우저에 잠깐 기억돼서 매번 안 쳐도 됩니다.

---

## 무료로 하고 싶다면 (주의)
Render **무료 인스턴스**도 가능하지만, 영구 디스크를 못 붙여서 서버가 잠들었다 깨거나 재배포되면 **입력 데이터가 초기화**됩니다.
- 임시/테스트용으로만 권장.
- 데이터를 지키려면: 가끔 **백업 내보내기** 버튼으로 `cozyflower-backup.json`을 저장해 두세요. (초기화되면 다시 불러오기로 복구)

## 직접 서버(VPS 등)에서 돌릴 때
```bash
# Node 18+ 설치된 서버에서
GUILD_PW="원하는비번" npm start
# 기본 포트 4321 (PORT 환경변수로 변경 가능)
# 데이터는 data/members.json 에 저장됨 (DATA_DIR 로 경로 변경 가능)
```
도메인/HTTPS는 Nginx 등 리버스 프록시로 연결하세요.

## 비밀번호를 바꾸려면
호스팅의 환경변수 `GUILD_PW` 값을 바꾸고 재배포(또는 재시작)하면 됩니다.
